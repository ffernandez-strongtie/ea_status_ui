# PM Review Questions

Use these questions during refinement before final commitment.

1. Should the Projects page remain a flat transmission list, or should the mockup's project-grouped browser become the target production behavior?
2. If project grouping is approved, what field defines a project group in production data?
3. Should users be able to view all transmissions across all projects at once after grouping is introduced?
4. What is the authoritative ordered list of transmission statuses?
5. Are `Send Components`, `Received Complete`, `Final Approval`, and `Ready for Pickup` all production statuses, or are some mockup-only labels?
6. Should transmission notes be implemented in this release, hidden, or treated as a future enhancement?
7. If notes are implemented, are they audit records, editable comments, or lightweight UI notes?
8. Should column visibility and column order persist per user?
9. Should filters persist across navigation, refresh, or sessions?
10. Which existing production fields must never be removed from Projects, Users, Groups, Jobs, and detail pages?
11. Should the Users list add role/group/status columns, or preserve production columns only?
12. Should Groups show status, or preserve production fields only?
13. What should delete behavior be for users and groups when records are referenced by existing transmissions or membership records?
14. Should mobile users get card-style records, horizontal tables, or a hybrid layout?
15. What is the minimum demo that proves the UI refresh is successful?
16. What would make stakeholders reject the refreshed UI after implementation?
17. Are there audit, legal, or compliance constraints for changing user/group management screens?
18. Should the implementation stay in the current production stack, or is a framework migration separately approved?
