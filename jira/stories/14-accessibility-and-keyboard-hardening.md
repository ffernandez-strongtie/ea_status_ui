# Story 14: Accessibility And Keyboard Hardening

## Value Statement

As a keyboard or assistive technology user, I want the refreshed StatusUI screens to be navigable and understandable so that I can complete the same workflows as mouse users.

## Details For Developers

Audit and harden app shell, navigation, menus, filter panels, dialogs, tables, actions, forms, badges, and status visualizations. Prioritize WCAG 2.1 AA practical compliance for the refreshed workflow.

## Acceptance Criteria

- Primary workflows can be completed by keyboard.
- Dialogs/flyouts use dialog semantics, focus trap, Escape close, and focus restoration.
- Icon-only controls have accessible names.
- Form fields have visible labels or accessible labels.
- Status colors are not the only way information is conveyed.
- Focus states are visible.
- Automated accessibility checks or documented manual checks are completed.
