# StatusUI - Engineering Status Service

## Overview
Simpson Strong-Tie "Engineering Status Service" admin dashboard. Single-page app with client-side routing (hash-based). No backend, no auth — all state is in-memory React.

## Pages
- Projects & Transmissions (index.html) — main table with search, filters flyout, notes flyout
- Users, Groups, Jobs — CRUD list pages
- Maintenance — system settings
- Account — profile page
- Detail/Form pages for users, groups, submissions

## Design Language
- **Brand:** Simpson Strong-Tie orange (`--sst-color-orange-400: #d97706`)
- **Status indicators:** Two patterns:
  - `StatusBadge` (dot + label) for Users, Groups, Jobs, and project list badges
  - `StageProgressBar` for transmission table — multi-segment progress bar showing workflow position. Stages: Send Components → Received Complete → 1st Review → 2nd Review → Final Approval → Ready for Pickup. Green = completed, orange = current, gray = remaining
- **Avatars:** Jira-style circles with initials, deterministic color from name hash. `Avatar` component supports `imageUrl` prop for future photo support
- **Tables:** Resizable columns with sort. `stickyColumns` prop freezes columns with scroll shadow. `reorderable` prop enables drag-to-reorder on non-sticky columns. Drag handle (⁞) shown on draggable headers
- **Notes:** Per-transmission notes with flyout pane (same sheet pattern as filters). Notes button with count badge in actions column. In-memory `NotesMap` state in `StatusApp`
- **Flyout panes:** Reusable `.sheet` / `.sheet-backdrop` pattern for filters and notes

## CSS Architecture
- Custom CSS in globals.css using CSS custom properties
- Key class families: `.status-light-*`, `.avatar`, `.sticky-col`, `.notes-*`, `.sheet`, `.card`, `.button`, `.field`
- Design system components used: `Calendar` + `Popover` for date pickers in filters (from `@/components/ui/`)
- Filter dropdowns use `.field` class (unified with text inputs), selects use custom chevron SVG background

## Filters
- Date fields: `DatePickerField` component using Calendar + Popover from design system
- Sender, Customer: dropdown selects populated from project data
- State, Status, Type, Priority: dropdown selects with `field` class styling
- Search bar accepts both project names and customer names

## Technical Notes
- Single component file: `app/components/status-app.tsx` (~2170 lines)
- Client-side routing via `usePathname()` hook reading `window.location.hash`
- Mock data constants: `PROJECTS`, `INITIAL_USERS`, `INITIAL_GROUPS`, `JOBS`
- `CURRENT_USER = "Frank Fernandez"`
- State lives in `StatusApp`: users, groups, notes
- Projects table columns order: Actions (sticky), Transaction ID (sticky), then data columns
- Each transmission row shows single US state abbreviation (CA, CO, TX, etc.)
